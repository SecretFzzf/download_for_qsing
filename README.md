# 🎵 歌曲下载器

一个功能完整的歌曲下载工具，支持多个音乐平台，智能分析页面并自动提取音频文件。

## ✨ 特点

- ✅ **智能爬虫** - 自动分析音乐网页，提取音频直链
- ✅ **前后端分离** - 前端HTML/JS，后端Python爬虫
- ✅ **支持多平台** - QQ音乐、网易云等
- ✅ **简洁界面** - 一键下载，操作简单
- ✅ **可靠下载** - 直接获取真实音频文件，不会损坏

## 🚀 快速开始

### 环境要求

- Python 3.7+
- Flask
- requests

### 安装依赖

```bash
pip install flask requests
```

### 运行项目

1. **启动后端服务器：**
```bash
python server.py
```

2. **打开浏览器访问：**
```
http://localhost:5000
```

## 📖 使用方法

1. **输入歌曲名称**
   - 例如：《三十岁的女人》
   - 这个名称会作为下载后的文件名

2. **输入歌曲网页链接**
   - QQ音乐：`https://kg.qq.com/node/play?s=...`
   - 网易云：`https://music.163.com/song?id=...`
   - 其他平台的歌曲详情页链接

3. **点击"下载"按钮**
   - 系统会自动分析页面
   - 提取音频直链
   - 下载到浏览器的默认下载位置

## 🔧 工作原理

```
用户输入网页链接
        ↓
后端爬虫获取页面HTML
        ↓
正则表达式提取音频直链
        ↓
下载音频文件
        ↓
返回给前端，在浏览器中下载
```

## ⚙️ 配置修改

### 修改监听地址和端口

编辑 `server.py` 最后的 `app.run()` 部分：

```python
# 默认：本地访问 http://localhost:5000
app.run(debug=False, host='127.0.0.1', port=5000)

# 公网访问：
app.run(debug=False, host='0.0.0.0', port=5000)
```

### 添加新的爬虫规则

编辑 `server.py` 中的 `@app.route('/api/download')` 函数，添加新的正则模式：

```python
patterns = [
    r'"playurl"\s*:\s*"(http[^"]+\.(?:m4a|mp3)[^"]*)"',  # QQ音乐
    r'(https?://[^\s"<>]+\.(?:m4a|mp3)[^\s"<>]*)',        # 通用MP3/M4A
    r'"src"\s*:\s*"(http[^"]+)"',                          # 其他格式
]
```

## 🔍 故障排查

### 问题1：显示"无法连接服务器"
- ✅ 检查后端服务器是否运行：`python server.py`
- ✅ 确保浏览器访问的是 `http://localhost:5000`

### 问题2：显示"未找到音频文件"
- ✅ 链接可能不支持（某些平台有防护）
- ✅ 检查链接是否有效
- ✅ 尝试其他音乐平台的链接

### 问题3：下载速度慢
- ✅ 这是正常的，取决于网络和服务器
- ✅ 较大文件可能需要等待

## 🙋 常见问题

**Q：为什么文件会损坏？**
A：新版本采用爬虫方式获取真实直链，不会再出现文件损坏问题。

**Q：支持哪些音乐平台？**
A：原则上支持任何能通过HTML提取音频URL的平台。主要测试了QQ音乐和网易云。

**Q：下载的文件可以商用吗？**
A：不可以，仅供个人学习。请尊重音乐创作者的版权。

**Q：可以在手机上使用吗？**
A：可以，部署后访问服务器地址即可。比如局域网内的其他设备可以访问 `http://服务器IP:5000`
